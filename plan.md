# Implementation Plan: WiFi + Volume → 统一 Settings 按钮

## Goal

将当前主页面上独立的 WiFi 按钮（右上角 730,0）和音量按钮（左上角 0,0）合并为一个统一的 **Settings 按钮**，点击后弹出设置面板，面板内通过页签或分区分列显示 WiFi 和音量两个子功能。

---

## 🎯 推荐方案：**EEZ Studio 重构（首选）**

> **核心理念**：在 EEZ Studio 中统一修改项目源文件，然后一键重新生成全部 C 代码。手写 C 代码（`actions/`、`vars/`）只需做接口层面的少量适配。

### 为什么必须用 EEZ Studio？

1. `screens.c` / `screens.h` / `ui.c` / `styles.c` 头部注明 `/* Autogenerated */`，手动修改会在下次 EEZ 重新生成时丢失
2. `assets[]` 二进制数组是序列化的 Flow Graph，手动改写几乎不可能
3. 对象的 `flowPropagateValueLVGLEvent` 索引、`evalTextProperty` 参数索引都编码在 assets 中
4. 如果不改 EEZ 项目，单纯写纯 C 覆盖层会导致大量废弃代码和对象引用断裂

## 📋 分步执行计划

### 阶段一：EEZ Studio 项目改造（源文件层）

#### 步骤 1：安装/确认 EEZ Studio 可用
- **内容**：确认有 EEZ Studio（https://github.com/eez-open/studio）可打开 `.eez-project`
- **验证**：能打开 `main/ui/eez/eez.eez-project` 并看到 main_page 可视化布局

#### 步骤 2：在 EEZ Studio 中重构 Widget 层级
- **当前结构**（需要改的地方）：
  ```
  main_page
  ├── obj0 (LVGL User Widget → wifi_btn 实现)
  │   引用了 User Widget "wifi_btn"
  └── obj1 (LVGL User Widget → volume_slider 实现)
      引用了 User Widget "volume_slider"
  ```
- **目标结构**：
  ```
  main_page
  └── obj_settings (LVGL User Widget → settings_btn 实现)
      引用了新的 User Widget "settings_btn"
      └── settings_panel (弹出面板，内含)
          ├── Tab1: WiFi 相关控件
          │   ├── wifi_list
          │   ├── wifi_refresh_btn
          │   ├── wifi_cancel_btn
          │   └── wifi_loader (spinner)
          └── Tab2: 音量相关控件
              ├── volume_slider
              └── volume_cancel_btn
  ```

- **具体操作**：
  1. 删除 `obj0` 和 `obj1` 两个 LVGL User Widget
  2. 新建一个 `LVGL User Widget`（命名为 `settings_btn`），放置在合适位置（建议居中偏上或右上角）
  3. 在 settings_btn 的 User Widget 定义中：
     - 创建一个 settings 入口按钮（settings_icon + 文字标签）
     - 创建一个弹出面板（500×400 左右）
     - 在面板内用 **TabView** 或独立容器做 WiFi/音量两个分区
     - WiFi 区域复用原来的 list、spinner、refresh_btn、cancel_btn
     - 音量区域复用原来的 slider、cancel_btn
  4. 配置 Flow 逻辑：
     - 按钮点击 → 动画显示/隐藏面板
     - 各子控件的 Value Changed 事件正确绑定 action
  5. 导出/重新生成 C 代码

- **风险评估**：中等 — EEZ Studio 的 LVGL User Widget 功能需确认对 TabView/多子控件嵌套支持程度
- **预估工时**：2-4 小时（取决于 EEZ Studio 熟练程度）

#### 步骤 3：确认生成代码的正确性
- **内容**：EEZ 重新生成后，对比新旧 `screens.c` / `ui.c` / `styles.c`
- **关注点**：
  - `objects_t` 结构体成员名称是否变为 `obj_settings__wifi_list` 等新名字
  - `object_names[]` 数组是否与新结构对应
  - `native_vars[]` 数组是否仍然保有 3 个元素（NONE, wifi_is_connected, volume）
  - `actions[]` 数组是否仍然是 `[action_wifi_start_scan, action_volume_change]`
  - `assets[]` 数组是否更新
- **验证**：编译通过 + UI 能正常初始化不崩溃

## 🅱️ 备选方案：纯 C 手动重构（不修改 EEZ 项目）

> **适用场景**：无法使用 EEZ Studio 或时间紧迫时采用。

### 核心思路
不修改 `.eez-project`，在 `screens.c` 的 `create_screen_main_page()` 中：
1. 隐藏原来的 `obj0__wifi_btn` 和 `obj1__obj2`（两个独立按钮）
2. 新增一个纯 LVGL 手写的 settings 按钮 + 面板
3. 复用手写 actions 文件中的 WiFi/音量逻辑

### 分步操作

#### 步骤 B1：在 `screens.c` 中添加 `create_settings_widget()` 函数
- **文件**：`main/ui/screens.c`（虽然自动生成，但手动添加不会被 EEZ 覆盖的函数）
- **内容**：
  ```c
  static void create_settings_widget(lv_obj_t *parent) {
      // 1. 创建设置入口按钮
      lv_obj_t *settings_btn = lv_button_create(parent);
      lv_obj_set_pos(settings_btn, 730, 0);  // 或更合适的位置
      lv_obj_set_size(settings_btn, 70, 40);
      add_style_user_btn(settings_btn);
      lv_obj_t *icon = lv_label_create(settings_btn);
      lv_label_set_text(icon, LV_SYMBOL_SETTINGS);
      add_style_user_icon_label(icon);
      
      // 2. 创建设置面板（初始隐藏）
      lv_obj_t *panel = lv_obj_create(parent);
      lv_obj_set_pos(panel, 150, -480);
      lv_obj_set_size(panel, 500, 440);
      lv_obj_add_flag(panel, LV_OBJ_FLAG_HIDDEN);
      
      // 3. 在面板内用 tabview 分区
      lv_obj_t *tabview = lv_tabview_create(panel);
      lv_obj_t *tab_wifi = lv_tabview_add_tab(tabview, "WiFi");
      lv_obj_t *tab_volume = lv_tabview_add_tab(tabview, "音量");
      
      // 4. WiFi 页：复用 wifi_actions.c 的逻辑（手动构建 list 等）
      // 5. 音量页：复用 volume_actions.c 的逻辑
      
      // 6. 点击按钮 → 动画显示面板
      lv_obj_add_event_cb(settings_btn, _settings_btn_click_cb, LV_EVENT_CLICKED, panel);
  }
  ```

- **风险**：
  - 添加在自动生成文件中的代码可能在某次 EEZ 重新生成时被覆盖
  - 可以单独新建 `main/ui/settings_widget.c` 独立文件来规避

#### 步骤 B2：将 WiFi panel 和 Volume panel 的 LVGL 对象引用改为手写创建
- **修改**：`wifi_actions.c` 中 `objects.obj0__wifi_list` 等引用需要改为全局变量或传入参数
- **新增全局变量**（在 `screens.c` 或独立文件）：
  ```c
  lv_obj_t *g_settings_wifi_list;
  lv_obj_t *g_settings_wifi_loader;
  lv_obj_t *g_settings_volume_slider;
  lv_obj_t *g_settings_volume_icon;
  ```

#### 步骤 B3：在 `create_screen_main_page()` 中隐藏旧的按钮
- **文件**：`main/ui/screens.c`
- **内容**：在 `create_screen_main_page()` 末尾添加：
  ```c
  lv_obj_add_flag(objects.obj0__wifi_btn, LV_OBJ_FLAG_HIDDEN);
  lv_obj_add_flag(objects.obj1__obj2, LV_OBJ_FLAG_HIDDEN);
  ```
  或在创建后立即隐藏

- **注意**：隐藏不等于删除，`objects_t` 结构体成员仍然存在且可被 actions 引用

### 备选方案评估

| 维度 | 评分 | 说明 |
|------|------|------|
| 实现速度 | ⭐⭐⭐⭐ | 3-5 小时可完成 |

## ⚠️ 风险清单

| # | 风险 | 严重度 | 缓解措施 |
|---|------|--------|----------|
| 1 | EEZ Studio 不支持嵌套 TabView + User Widget 组合 | 高 | 降级为简单容器+手动切换可见性；或采用方案二 |
| 2 | EEZ 重新生成后 `native_vars[]` 顺序改变，导致 WiFi/音量变量索引偏移 | 中 | 重新生成后立即比对 `ui.c` 中的 `native_vars[]`，如果在 EEZ 中保持定义顺序不变则无影响 |

| 4 | 旧面板动画的 flowPropagateValueLVGLEvent 索引在重构后失效 | 中 | EEZ 重构时会自动重新计算，但需确认新的 button→panel 动画链正常 |
| 5 | 密码输入 UI（`wifi_actions.c`）创建在 `lv_screen_active()` 上，如果 settings panel 是 overlay 可能遮挡 | 低 | 将密码输入容器创建在 settings panel 内部 |

| 7 | 两个方案都可能在触摸事件处理上有冲突（panel 外点击穿透） | 低 | 给 panel 添加 `LV_OBJ_FLAG_CLICKABLE` 并设置半透明遮罩背景 |

## 🔧 建议的最终方案路径

1. **先尝试 EEZ Studio**（方案一）— 这是架构正确的做法
2. **如果 EEZ Studio 限制太多**（如不支持 TabView + User Widget 组合），退而求其次：
   - 在 EEZ 中只做一个简单的 settings 按钮 + 空面板容器
   - 面板内部的 WiFi/音量列表、滑块等由手写 C 代码动态填充（混合方案）
3. **如果 EEZ Studio 完全不可用**，采用方案二纯 C 代码

## ✅ 验收标准

1. 主页面只显示一个 Settings 按钮（不再有独立的 WiFi 和音量按钮）
